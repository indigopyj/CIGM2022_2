The width of the image is 4290, and the height is 2856 according to the code above. And I figured out that the image has 16 bits per unsigned interger by checking the image’s data type(dtype). Therefore, I converted the image into a double-type array by ‘.astype(’double’)’.

LINEARIZATION [5 pts]

Before applying linear transform, the maximum and the minimum values within the image were 15303 and 2002. I applied a linear transform by shifting an image with 2047 and scaling it with (15000 - 2047), so that the value 2047 is mapped to 0, and the value 15000 is mapped to 1. After a linear transform(shift/scale), the maximum and the minimum values were 1.023 and -0.003. Then, I clipped negatives values to 0 and values greater than 1 to 1 by ‘numpy.clip’.

IDENTIFYING THE CORRECT BAYER PATTERN [20 pts]

In order to find the bayer pattern, I applied white balancing and demosaicing to all kinds of patterns first and then checked which image is printed as normal. White balancing and demosaicing will be explained in details in the next section. Finally, I found that ‘RGGB’ is the right patterns since only ‘RGGB’-patterned image has natural colors.

The code for converting bayer pattern to RGB channels is as follows:

You need to filter only necessary pixels in each channel.

For example, assume that we consider converting ‘RGGB’ pattern to RGB channels.

For channel ‘R’, select the pixels in the array between 0th index and the end of the row every 2 step along the x-axis and between 0th index and the end of the column every 2 step along the y-axis.

For channel ‘B’, select the pixels in the array between 1th index and the end of the row every 2 step along the x-axis and between 1th index and the end of the column every 2 step along the y-axis.

However, for channel ‘G’, the process is slightly different, since green pixels have two different patterns:

  1. Select the pixels in the array between 1th index and the end of the row every 2 step along the x-axis and between 0th index and the end of the column every 2 step along the y-axis.
  1. Select the pixels in the array between 0th index and the end of the row every 2 step along the x-axis and between 1th index and the end of the column every 2 step along the y-axis.

Then, gather the filterd pixels for each channel and stack them along the color dimension.

Finally, we can get the RGB-channel image.

WHITE BALANCING [20 pts]

I found that ‘RGGB’ is the right pattern by the above section, so I changed RGGB 1-channel to RGB 3-channels by a function ‘bayer_to_rgb’. With RGB image, I applied white balancing with two options.

For gray world automatic white balancing, I computed per-channel average and normalize each channel by its average. Then, I multiplied them by green channel average.

For white world automatic white balancing, I computed per-channel maximum and normalize each channel by its maximum. Then, I multiplied them by green channel maximum.
